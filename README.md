# PYTHON CODE FOR SNAKE GAME
The game should attempt to include the following features: <br />
1. The use of images. <br />
2. The use of shapes. <br /> 
3. The use of text. <br />
4. A scoring mechanism. <br />
5. A leader board which is presented at appropriate places in the game (i.e. from a menu before
the game begins and/or at the end of the game) <br />
a. The leader board must retain player names (or initials if you adopt a retro style), with their
score and their position in the leader board.<br />
b. When the game is quit and reloaded, the leader board will reflect the history of the leaders
of the game.<br />
6. The movement of objects. <br />
7. The ability for the user to move an object.<br />
8. Some form of collision detection.<br />
9. The ability for the player to pause and unpause the game (so that they can make a cup of tea).<br />
10. The ability to customise the experience for the user, for example, if the game has an object that
can be moved the player should be able to define the keys that control that object.<br />
11. Save/Load game feature so that the player can resume playing where they left off tomorrow. <br />
12. A ‘boss key’, we shouldn’t play games at work. Some games introduced a boss key which allowed
the game to flip to an image that gave the impression that the player was doing something work
related quickly.
